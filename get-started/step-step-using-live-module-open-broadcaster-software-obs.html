一步步：將實時模塊與Open Broadcaster Software（OBS）一起使用

這個快速入門的目的在於向您展示如何使用視訊雲端直播模組和開放廣播軟體 (OBS) 來廣播現場活動。
 

要了解實時模塊中可用的所有功能，請參閱使用實時模塊創建和管理實時事件。有關有助於確保高質量，穩定的實時流媒體體驗的一系列最佳做法和建議，請參閱實時模塊準則和最佳實踐。有關Brightcove提供的用於傳輸實時流視頻的所有選項的列表，請參見傳送直播。

筆記：
  • 有關使用Telestream Wirecast編碼器的本文檔版本，請參閱一步步：將實時模塊與Telestream Wirecast一起使用。
  • 實時流支持僅適用於Video Cloud Enterprise發布者。

完成此「快速入門」後，您應該能夠：

  • 使用視訊雲端直播模組建立即時活動
  • 為即時事件設定您的編碼器
  • 修剪您的活動，並將其儲存為視訊雲端帳戶中的 VOD 資產

這個快速入門將幫助您廣播現場活動使用開放廣播軟件（OBS）Studio 不同的設備。OBS是用於視頻錄製和實時流傳輸的開源軟件，可以將在這裡下載。其他編碼器也可以與 Live 模組搭配使用。

受眾

任何需要使用影片雲端直播模組播放即時串流活動的 Video Cloud 發行者。

先決條件

  • 可訪問實時模塊的Video Cloud Enterprise帳戶
  • 播放活動的攝影機
  • 您必須擁有自己的編碼軟件。本快速入門使用OBS工作室可以免費下載（請參閱支持的編碼器）

限制

  • 您的編碼器必須輸出 RTMP 或 SRT 串流
  • 直播將經歷 20-30 秒的延遲
  • 串流必須在活動開始後 30 分鐘內開始 (預設層級帳戶設定)

直播活動的步驟

若要使用直播模組串流您的即時活動，您將完成下列工作：

  • 使用Video Cloud Live模塊創建新事件
  • 配置編碼器
  • 發布實時流
  • 從現場直播中創建片段

建立新的直播活動

  1. 在瀏覽器窗口中，導航到Video Cloud Studio。
  2. 輸入您的Video Cloud登錄憑據，然後單擊登入。
  3. 目前的 Video Cloud 帳戶名稱會出現在頁面的右上角。如果您有多個帳戶，請按一下帳戶選擇器下拉式清單，然後選取您要儲存實況資產的帳戶。如果您選取的帳戶尚未啟用 Live，即時模組將會以試用模式開啟。
  4. 點擊居住在導航標題中。
  5. 點擊建立活動按鈕創建一個新事件。
  6. 輸入一個活動名稱。
  7. 輸入一些標籤為事件。
  8. 如果您的帳戶已設定必要的自訂欄位，請確定已指派自訂欄位值。
  9. 選擇一個實時攝取資料從下拉列表中。即時內嵌描述檔可控制所建立的即時轉譯的數量和品質。
  10. 選擇完成後將事件轉換為視頻資產。這會將事件儲存為媒體模組中的視訊。
  11. 選擇地區最接近您的位置。
  12. 點擊高級選項鏈接以展開高級選項。
  13. 設置重新連接時間到10分鐘（默認）。
    OBS Encoder 支持 SRT 協議，您可以在輸入格式的部分高級選項菜單。
  14. 點擊開始流式傳輸。Live模塊將返回一個流端點（RTMP URL）和流名稱。這些值將用於編碼器設定中。
    如果您為直播活動選擇了 SRT 協議，則可以在 OBS 中通過在流設置服務下選擇“自定義”，然後輸入您的srt://服務器框中的路徑。

設定您的編碼器

在本節中，我們將配置和使用 OBS Studio 編碼器來支持即時串流事件。OBS Studio 軟件是一個桌面應用程序，它從你的攝像機捕獲輸入，並產生一個可以由 CDN 交付的流。請注意，有些硬體和其他軟體的編碼解決方案可能更適合用來傳遞您的直播活動。您可以下載OBS Studio軟件這裡。

筆記：
  • 在繼續之前，您應該已經安裝了OBS軟件並已將相機連接到計算機。
  • 這個例子使用了使用黑魔術設計超級工作室迷你錄音機連接到筆記本電腦的數碼單鏡反光相機。
  • OBS 提供了記錄流的本地副本的能力。如果選擇這樣做，您將擁有串流的本機複本，您可以建立剪輯片段，以防事件廣播期間發生任何網路問題。有關如何記錄本地文件的信息，請參見OBS文檔。
  1. 開啟「OBS 工作室」。
  2. 在裡面資料來源部分，單擊+添加新的來源。選擇一個來源從列表中。此快速入門使用通過Blackmagic Design UltraStudio Mini Recorder連接到筆記本電腦的DSLR，因此黑魔法裝置被選中。
  3. 輸入新來源的名稱，然後單擊好的。
  4. 從中選擇您的相機設備列表。預覽應該會出現在屬性對話方塊中。點擊好的。
  5. 在控件部分中，單擊設定值。
  6. 點擊溪流在“設置"對話框的左側。
  7. 為了流類型選擇自定義流服務器。
  8. 對於網址，請使用流URL由實時模塊提供。
  9. 為了流鍵， 用活。
  10. 點擊好的保存設置。
  11. 在控件部分中，單擊開始流式傳輸。現場直播應該開始。
筆記：處理您的流大約需要一分鐘。

發佈現場活動

如果要產生即時事件的內嵌程式碼，請依照下列步驟執行：

  1. 返回直播模組。
  2. 如果未顯示事件的“控制室"頁面，請單擊居住標籤以顯示實時事件列表。按一下事件名稱以開啟 [ 控制室] 頁面。
  3. 確認您可以看到即時直播。請注意，啟動編碼器後，可能需要一分鐘才會顯示即時串流。
  4. 點擊發布和嵌入。
  5. 選擇一個播放器然後點擊播放器網址觀看直播。
  6. 直播結束後，請點擊停止流實時模塊中的按鈕。此外，返回 OBS Studio 並單擊流按鈕，停止編碼器從流媒體。
    筆記：當直播完成時，請務必關閉/停止編碼器。使任何編碼器和 Brightcove 之間的連接保持打開狀態，即使沒有發送數據，也可能導致意外的連接費用。

建立剪輯片段

Live 模組提供從已完成的事件建立視訊剪輯的功能。您也可以在現場活動進行時建立剪輯片段。建立剪輯片段的原因有幾種：

  • 建立可張貼至社交媒體的簡短、傳情剪輯
  • 限制您要張貼的影片長度 (大多數平台會限制視訊的長度)
  • 您發現較短的視頻在社交媒體上表現更好

剪輯片段只能在活動結束後 7 天內建立。之後，創建剪輯按鈕將被禁用。如果事件超過 24 小時，只有過去 24 小時可以剪輯。

筆記：有關使用剪輯界面的完整詳細信息，請參見使用直播模塊創建視頻剪輯.

若要建立剪輯片段，請依照下列步驟執行：

  1. 返回直播模組。
  2. 點擊完全的標籤。
  3. 找到事件並按一下事件名稱以開啟控制室。
  4. 點擊創建剪輯。剪裁編輯器將打開。
  5. 單擊並拖動剪輯手柄 () 在焦點時間線上設置剪輯的起點和終點。
  6. 點擊預覽剪輯預覽剪輯。可以根據需要調整起點和終點。
  7. 點擊保存剪輯。
  8. 進入一個剪輯名稱和其他元數據值。帳戶的所有必要欄位也會出現，且應該被指派值。
  9. 點擊創建剪輯。擷取程序將會開始，剪輯片段會儲存至您的 Video Cloud 帳戶。
  10. 剪輯片段會出現在剪輯片段編輯器下方。點擊發佈到社交使用Brightcove Social將剪輯發佈到您的社交目的地之一。有關使用“社交"模塊發布視頻的信息，請參閱將視頻發佈到社交媒體平台。
  11. 點擊返回控制室：完全的麵包屑。所有剪輯片段也會出現在活動的「控制室」中。
    筆記：您可以多次按一下「建立剪輯片段」按鈕，建立多個廣播片段。這允許您根據需要為每個廣播修剪和保存任意次數，將剪輯保存為媒體庫中的單獨 VOD 資產。

即時串流問題疑難排解

如果您需要進一步的幫助以使現場活動開始進行，可以聯繫我們。為了確保您得到最快的回應，以下是解決問題所需的支援清單。

  • 流的具體症狀。例如，它是根本不播放還是口吃或凍結？
  • 這個流是否在過去正常工作
  • 您在編碼器中使用的入口點 URL
  • 您使用的編碼軟件和硬件
  • 您發佈直播活動的玩家 URL
  • 影片雲端工作室中即時資產的視訊 ID
  • 從編碼器到發佈點主機的追蹤路由結果
