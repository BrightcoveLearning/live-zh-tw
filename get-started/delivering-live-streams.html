提供實時流

本主題說明 Brightcove 提供即時串流的選項。

Video Cloud 即時串流視訊功能可讓您提供高品質的即時直播廣播至任何裝置或瀏覽器。您可以在沒有任何自訂開發或複雜整合的情況下提供即時串流，但是如果您有自訂需求或需要自訂整合，則可使用 API。

筆記：實時流媒體支持僅適用於 Video Cloud Enterprise 發布商。

視訊雲端也支援使用 DVR 即時串流。當Brightcove Player配置有DVRUX插件，實時流的行為就像通過觀眾的數字視頻錄像機播放一樣。

Brightcove 提供多種提供直播的選項。本主題包含下列各節：

  • 硬件與基於雲的編碼
  • 使用基於軟件的編碼器進行簡單的直播
  • 使用Video Cloud Live模塊進行實時流式傳輸
  • 使用基於硬件的編碼器進行實時流式傳輸

硬體與雲端式編碼

布萊特灣支援硬體和雲端式編碼。各種因素可能會決定硬體還是雲端式編碼最適合事件。

何時使用硬體編碼

  • 24x7 線性饋送，其中容量已知
  • 以單一位置為基礎的架構
  • 廣播格式支援

何時使用雲端編碼

  • 針對事件
  • 容量需求未知時
  • 適用於去集中式組織
  • 事件時間未知且速度很重要時
  • 如果有頻寬瓶頸

使用軟體式編碼器進行簡單的即時串流

此選項使您可以使用Akamai Stream Packaging和 基於軟件的編碼器，用於將直播事件流式傳輸到Brightcove Player。有關更多信息，請參見使用Akamai流打包和Telestream Wirecast創建實時流。

使用視訊雲端直播模組即時串流

Video Cloud Live 模組提供簡單易用的介面，可協助您在桌上型電腦和行動裝置上直播活動。不需要編程。在直播模組中輸入您的活動資訊後，系統會提供一組編碼器設定，讓您插入現場編碼器。所有轉碼都在雲端完成，因此您不需要基於硬件的轉碼器。如需詳細資訊，請參閱其中一份文件。

  • 使用實時模塊創建和管理實時事件 -提供有關使用Video Cloud Live模塊的所有詳細信息
  • 一步步：將實時模塊與Telestream Wirecast一起使用 -提供有關將Video Cloud Live模塊與Telestream Wirecast結合使用的分步指南
  • 一步步：將實時模塊與Open Broadcaster Software（OBS）一起使用 -提供有關將Video Cloud Live模塊與Open Broadcaster軟件一起使用的逐步指南
  • 實時模塊準則和最佳實踐 -提供一系列最佳做法和建議，以幫助確保高質量，穩定，實時的流媒體體驗

即時 API

即時 API 是以 REST 為基礎的 API，可讓您建立和管理即時串流事件。可選功能包括：

  • 伺服器端廣告插入 (SSAI)
  • AES-128 加密
  • 從即時串流中擷取的剪輯片段建立隨選視訊資產
  • DVR 功能
  • 多個 CDN

有關更多信息，請參見實時API文檔。

使用硬體式編碼器進行即時串流

此選項使用硬體式編碼器編碼，並將即時串流傳送至 CDN。在 Video Cloud 端，您可以建立遠端資產來託管串流，然後您可以發佈播放器進行即時串流。
