創建自定義實時攝取配置文件

在本主題中，您將學習如何建立和管理自訂即時內嵌設定檔。

「內嵌設定檔管理」頁面可用來管理自訂即時內嵌設定檔。即時內嵌描述檔會決定 Brightcove Live 所建立的轉譯數量和品質。將影片上傳至 Video Cloud 帳戶時，不會使用即時內嵌描述檔。內嵌設定檔是使用 JSON 定義的。在創建新的攝入資料之前，請查看標準實時攝取配置文件文檔，以查看是否有現有的配置文件可以滿足您的需求。

筆記：
  • 每個輪廓最多可以建立 25 個轉譯。
  • 你能夠為垂直視頻創建字幕-僅包括height和width演繹規範中與視頻的長寬比匹配的值（1080像素是height）。
  • 當實時配置文件隱藏在Studio中的行政 > 攝取個人資料，這些將不會在建立現場活動 UI，但仍可以與Live API一起使用。
  • 直播等級：
    • 對於要被視為第 1 層的實時攝取配置文件，其所有再現的比特率總和不能大於 5450。
    • 如果其所有再現的比特率總和大於 5450 但小於 10000，則 Live 攝取配置文件將被視為第 2 層。
    • 如果其所有再現的比特率總和大於 10000，則該配置文件將被視為第 3 層。
每個配置文件最多可以創建 25 個再現。

建立新設定檔

「內嵌設定檔管理」頁面可用來建立自訂「即時內嵌」設定檔。請注意，要使配置文件出現在實時模塊用戶界面中，它應該具有"model_version": 1作為配置文件屬性的一部分。

若要建立新的自訂「即時內嵌」設定檔：

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊攝取個人資料關聯。
  3. 點擊新個人資料>實時。
  4. 進入一個名稱為配置文件，然後單擊創造。
  5. 依需要使用「紀要編輯器」編輯紀要。
  6. 點擊保存。

複製設定檔

您可以透過複製現有的設定檔來建立新的設定檔。然後，您可以使用您的變更來編輯新的設定檔。請注意，要使配置文件出現在實時模塊用戶界面中，它應該具有"model_version": 1作為個人資料的一部分。

若要透過複製現有的設定檔來建立自訂「即時內嵌」設定檔：

  1. 點擊行政 Studio標頭中的鏈接。
  2. 點擊攝取個人資料關聯。
  3. 用類型的標准直播複製。按一下設定檔名稱以開啟設定檔編輯器。
  4. 點擊複製。設定檔編輯器將會開啟新的設定檔。
  5. 捲動至設定檔的頂端。編輯name元素，為配置文件賦予有意義的名稱。這個名字將視頻雲工作室內顯示。
  6. 編輯description元素，為配置文件提供有意義的描述。
  7. 視需要編輯轉譯資訊。
  8. 點擊保存。

新建立的設定檔會出現在設定檔清單中。自定義配置文件可以通過單擊刪除 (） 按鈕。
