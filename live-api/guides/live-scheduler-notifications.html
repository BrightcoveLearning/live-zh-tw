簡介

完成任務後，調度器將使用 http 方法向配置的端點發送通知郵政詳細說明已執行的操作及其成功/失敗。

您還可以配置即將執行的任務的預通知。見實時API參考有關指定通知URL和設置的詳細信息，並註意在安排SEP作業的開始/停止時間和安排剪輯方面的區別：

啟動/停止調度程序
  • 通知URL在請求正文對象的頂層定義
  • 通知安排在激活和停用對象（事件發送通知之前的時間（以秒為單位））

例子

{
	"job_id": "your Job ID",
	"activation": {
		"time_utc": 1612274100,
		"notification": 600
	},
	"deactivation": {
		"time_utc": 1612281600,
		"notification": 600
	},
	"description": "My Tuesday broadcast",
	"notification_url": "http://example.com/live/jobstartstop/callbacks"
}
剪輯排程器
  • 通知URL在請求正文對象的頂層定義
  • 通知安排在夾子對象（任務開始發送通知之前的時間（以秒為單位））

例子

{ "job_id": "你的工作 ID", "描述":“我的預定剪輯”， "notification_url": "http://example.com/live/clip/callbacks"， “夾子”：{ "output": { "label": "My Scheduled Clip", "start_time": 1612878900, "end_time": 1612879800, "videocloud": { "video": { "name": "My Scheduled Clip" } } }, "time_utc": 1612879800, “通知”：600  }}

通知主體

當服務器向您發送通知時通知網址，請求將包含一個 JSON 正文。下表中詳細說明了這些字段。

通知正文字段
行動 動作名稱。看動作
工作流程 動作完成後更新了工作流程。結構將取決於工作流類型。
錯誤 如果在完成操作時發生錯誤，則此字段將包含錯誤。值取決於操作類型。

動作

下表按工作流類型詳細說明了操作。

動作
名稱 工作流程類型 描述
notify-activate jobstartstop 通知您您的工作即將被激活的通知。
activate jobstartstop 作業激活完成。如果“錯誤”存在，激活失敗，取消激活將被取消。
notify-deactivate jobstartstop 通知您您的工作即將被激活的通知。
deactivate jobstartstop 作業停用完成。如果“錯誤”存在，則取消激活失敗。
notify-clip clip 提醒您將很快請求您的Clip的通知。
clip clip 剪輯請求已完成。如果“錯誤”存在，請求失敗。
