Brightcove 現場直播：最佳實踐

本主題提供使用即時 API 建立即時串流的最佳作法指南。

概述

Brightcove 直播提供了一個強大的服務，用於創建直播活動或全天候直播。本指南概述最佳化直播的最佳做法

輸入頻寬

提供高品質、穩定的輸入串流是確保觀眾獲得最佳使用者體驗的唯一途徑。良好的輸入串流以最高一致的可用頻寬提供最佳的視訊品質。

  • 最小輸入頻寬：2.5 兆
  • 最大輸入頻寬：10 兆

支援的編碼器

看支持的直播活動編碼器有關已知可與Live一起使用的編碼器的列表。請注意，其他編碼器也可以工作，但尚未經過測試。

支援的 CDN

  • 阿卡
  • 亞馬遜雲端
上面的 CDN 得到官方支持，但其他基於文件的 CDN 應該可以工作。

重試

建議您從編碼器啟用 RTMP 連線的重試次數。5 秒的重試間隔進行大量的重試次數，將減輕編碼器與進入點之間的任何間歇性連線問題。

常見問題

建立即時工作後，您需要多久才能開始串流？
在Brightcove Live中，當狀態從waiting 至finishing ：
  1. 如果工作在等待狀態（尚未開始）和max_waiting_time_ms已過去，作業已完成/已停用。
  2. 如果工作在斷開連接狀態（已啟動，但已斷開連接），並且reconnect_time已過去，作業已完成/已停用。

如果event_length 大於30分鐘，作業將在30分鐘後終止。如果event_length 少於30分鐘，作業將在event_length 。

例如，如果event_length 是60分鐘，那麼實時作業將在30分鐘後終止。如果event_length 是15分鐘，那麼實時作業將在15分鐘後終止

這reconnect_time 對等待狀態沒有影響。

並發現時 job_設置有什麼限制？

最多5個活動等待，未開始任何時候都允許工作。

並行工單的其他限制：

  • 的數量channel（24x7）作業限制為0個或每個區域的數量較少（取決於帳戶類型）。
  • 並發數跑步event職位受地區限制，通常為100個。
  • 並發數等待連接event職位數限制為5。
  • 每個區域的SEP作業數量限制為3或10（請參見支持的AWS區域）。

任何這些限制都可以由支援在帳戶層級調整。如果您需要額外容量，請連絡您的客戶經理。

只要輸入頻寬足夠，布萊特灣直播能否推動 1080p 品質？
是，所有帳戶都啟用 1080p 輸入功能。
DRM 是否可用？
是的！如果您有興趣在真實帳戶中新增 DRM 支援，請聯絡您的客戶經理。
