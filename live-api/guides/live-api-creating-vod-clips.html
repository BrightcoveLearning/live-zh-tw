實時 API：創建 VOD 剪輯

本主題向您展示如何從實時流中創建視頻點播（VOD）剪輯。

概述

剪輯是從實時流中提取的視頻。它們可以發送到 S3 存儲桶、FTP 站點或視頻雲帳戶。剪輯將創建為MP4視頻，並且在所有情況下都是發送到目的地的內容。如果是視頻雲，MP4 將由攝取系統轉碼，為視頻創建的再現類型將取決於所使用的攝取配置文件。

剪輯的定義是使用/vods端點。

剪輯可以通過以下幾種方式創建：

  • 和stream_start_timecode和/或stream_end_timecode在SMPTE時間碼中為直播事件定義的- 請注意，這要求編碼器發送時間碼信息
  • 和start_time和/或end_time相對於開始時間的定義（stream_start_time）的整個直播活動
  • 和start_time和/或end_time以紀元（Unix）時間定義（以秒為單位）
  • 和duration
  • VOD API 能夠用於加密或受DRM保護職位。當前，Live Module不支持此功能，但在將來的版本中將支持。

注意事項

  1. 為了使片段盡快可用，首先創建了一個片段精確的片段，然後在可用時立即用一個幀精確的片段替換。
  2. 如果指定duration，結果片段將如下所示：
    • 如果作業處於活動狀態並且仍然有效：（請求時間-持續時間）至（請求時間）
    • 如果作業完成：（finished_at -持續時間）至（finished_at）
  3. 如果同時指定start_time和end_time：
    • 如果作業處於活動狀態並且仍然有效：只要大紀元時間窗口完全落在created_at和請求時間，將製作剪輯
    • 如果作業完成：只要Epoch時間窗口完全落在created_at和finished_at，將製作剪輯
  4. 使用的實時流剪輯智囊團將不包含廣告。
  5. 活動結束後最多7天即可創建剪輯。為了九月，它們可以創建到下一次激活或 7 天（以較短者為準）。
  6. VOD API不會在流中添加任何內容。如果在300秒長的實時流中指定350，則輸出將為300秒長。
  7. 您不必使用啟用DVR的直播就可以進行剪輯，因為直播是在廣播時存儲的，並且可以在事件結束後的7天內立即使用。
  8. Brightcove Live剪輯只會產生與高分辨率輸出相同分辨率的剪輯。它將不匹配源輸入分辨率（除非與最高分辨率輸出相同）。

剪輯也可以發送到多個目的地：

  • 一種視頻雲帳戶
  • FTP服務器
  • 一個S3存儲桶

當您指定剪輯時，輸出必須包含任何一個一種url目的地或者一種videocloud對象詳細說明視頻的創建和剪輯的攝取視頻雲 .

注意：短片可以在直播流運行時創建。為此，您需要以紀元時間或相對於時間定義剪輯的開始和結束時間開始直播時間。

證書

如果要將剪輯發送到的目標位置需要憑據才能訪問，則可以使用Live API的憑據操作來創建這些憑據。看管理Live API的憑據更多細節。

請注意，如果您將剪輯發送到與 Live 帳戶關聯的 Video Cloud 帳戶，您應該不是在裁剪作業中包含憑據。

示例1：相對於流開始的開始/結束時間

請求主體包括開始和結束時間，以及有關將剪輯發送到何處的詳細信息。這是一個示例請求正文，它創建流的第三分鐘的剪輯並將其發送到視頻雲帳戶：

{
  "live_job_id":"PUT-LIVE-JOB-ID-HERE",
  "outputs":[
    {
      "label": "60 secs by stream from min 2 to min 3",
      "stream_start_time": 120,
      "stream_end_time": 180,
      "credentials": "USER_VIDEOCLOUD_CREDENTIAL_LABEL",
      "videocloud": {
        "video": {
        "name": "One Minute Clip",
        "tags": ["live", "clip"]
        },
          "ingest": {
            "capture-images": true
        }
      }
    }
  ]
}

在這個例子中，我們正在創建一個持續時間為一分鐘的剪輯並將其發送到視頻雲 .我們給剪輯一個名稱和幾個標籤，而不是指定攝取配置文件用於重新轉碼，以便使用帳戶默認值，並指示視頻雲在轉碼期間從剪輯中捕獲縮略圖和海報圖像。

          "name": "One Minute Clip",
          "tags": ["live", "clip"]
          },
            "ingest": {

在此示例中，我們將在特定的紀元時間（本例中為2018年1月22日格林尼治標準時間08:24:54）創建一分鐘持續時間的剪輯。

請求正文-S3

請求主體包括開始和結束時間，以及有關將剪輯發送到何處的詳細信息。這是一個示例請求正文，它創建流的第三分鐘的剪輯並將其發送到S3存儲桶：

{
  "live_job_id":"{{job_id}}",
  "outputs":[
    {
      "label": "last_30",

      "credentials": "s3-credentials",

在此示例中，我們將創建一個30秒持續時間的剪輯並將其發送到S3存儲桶。我們提供存儲桶URL，其中包括剪輯的文件名，以及一個字符串，它是保存的S3存儲桶憑證的名稱-Brightcove支持人員可以為您的帳戶設置憑證。
