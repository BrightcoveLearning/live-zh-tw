概述

服務器端廣告插入(SSAI)允許您在直播活動期間的指定時間展示廣告。有關一般信息，請參見實時API：服務器端廣告插入(SSAI)文檔。

  • 在您可以使用之前SSAI，必須為您的帳戶啟用它。如需詳細資訊，請聯絡您的客戶經理

提示點

廣告休息時間由提示點觸發，可以通過兩種方式指定：

  • 編碼器發送給Brightcove
  • 通過創建的即時提示點實時 API

從編碼器

Brightcove實時交付系統可以解釋編碼器以AMF格式提交的提示點：

  AMFDataList
  [0]:onCuePoint
  [1]:{Obj[]:
    time: 1.9889, //Difference from PTS of THIS packet to the first PTS of the 1st video frame in the adbreak
    name: "scte35",
    type: "event",
    ad_server_data: "YWJjZGVmZ2g=",	// optional introduced by Brightcove. It is a base64 encoded json map of parameters e.g. {‘key’:’value’}
    parameters: {Obj[]:
      type: "avail_in",
      duration: 12.0
    }
  }

注意事項：

  • 僅有的avail_in RTMP輸入當前支持類型提示點。
  • RTP和SRT輸入支持SCTE-35提示點。
見下面的附錄有關 Elemental 編碼器的示例提示點設置的屏幕截圖。

手動插入提示點

您可以使用實時 API通過發送一個POST要求：

方法 POST
網址 https://api.bcovlive.io/v1/jobs/Job_ID/cuepoint
標頭 X-API-KEY: your API KEY

包括一個指定以下內容的請求主體：

欄位 類型 描述
duration 整數 休息時間（以秒為單位）。

插入提示點的持續時間需要為至少是段長度的兩倍就職（於）;著手（於。見持續時間的例子。
timecode SMPTE格式 可選的：SMPTE格式的時間碼， HH：MM：SS：FF（FF =幀），以指定何時應將一組任何變量（鍵/值對）傳遞給adServer。

如果省略，將立即插入提示點。

如果您使用時間碼屬性，則編碼器必鬚髮送 SMPTE 格式的 ( HH:MM:SS:FF ) 時間碼存儲在tc財產通過OnFI .時間碼是從直播開始的。
ad_server_data 物件 可選的：您傳遞的鍵/值對將取決於您使用的廣告服務器。有關更多詳細信息，請參見您的廣告服務器文檔和使用廣告宏定位廣告部分。

信標

信標是發送到第三方分析的播放數據點，以跟踪是否播放了廣告以及播放了多少廣告。在本節中，我們將查看可以使用實時 API，以及可用於提供數據的變量。下一部分將詳細介紹用於創建和管理信標集的API請求。

信標類型

信標類型
信標類型 描述
Load 每個會話觸發一次，僅在請求頂級清單時觸發
Play 已請求內容，並且返回了第一段
Heartbeat 目標持續時間（段秒）
AdStart 個別廣告已開始
AdFirstQuartile 前四分位（25％）
AdMidpoint 第二個廣告四分位（50％）
AdThirdQuartile 第三四分位（75％）
AdComplete 個別廣告已完成
AdBreakStart 廣告中斷已開始
AdBreakComplete 廣告中斷已結束

SCTE-35廣告變量

這電纜電信工程師協會（SCTE）定義了用於實時流的動態廣告插入的標準。下表總結了SCTE-35廣告配置變量。

帶有或不帶有前向糾錯（FEC）和基於傳輸流（TS）的安全可靠傳輸（SRT）輸入源的實時傳輸協議（RTP）當前都支持SCTE-35 ad參數。請注意，強烈建議將FEC用於RTP。

這些廣告參數不適用於使用實時消息傳遞協議 (RTMP) 的輸入流。
SCTE-35廣告配置變量
多變的
描述
scte35_eventID
隨SCTE35消息傳遞的唯一事件ID。
scte35_programID
隨SCTE35消息傳遞的唯一程序ID。
scte35_availNum
廣告可用的特定拼接時間的ID，通過SCTE35消息發送。
scte35_breakDuration
廣告中斷的中斷時間（以程序的90 kHz時鐘的滴答度表示）隨SCTE35消息傳遞。
scte35_spliceTime
廣告中斷的拼接時間（以程序的90 kHz時鐘的滴答度表示）隨SCTE35消息一起傳遞。

創建信標集

要創建信標集，請發送一個POST要求：

方法 POST
網址 https://api.bcovlive.io/v1/ssai/beaconsets
標頭 X-API-KEY: your API KEY

樣品索取主體

  {
    "account_id": "User's Account ID [Optional]",
    "beacon_urls": [
      {
        "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/load?position=load&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",
        "beacon_type": "Load"
      },
      {
        "beacon_url": "https://myserver.com/beaconRX/{{job.job_id}}/play?position=play&sid={{session.session_id}}&jid={{job.job_id}}&app={{application_ad_configuration.description}}&rnd32={{random.int32}}&rnd64={{random.int64}}&bid={{random.uuid}}&t={{server.timestamputc}}&ua={{client.useragent}}&ip={{client.ipaddress}}&ref={{client.referrer}}&ref={{client.referer}}&ab={{live.adbreak}}&abd={{live.adbreakduration}}&abdi={{live.adbreakdurationint}}",

方法 GET

刪除信標集

最後，要刪除信標集，請發送一個DELETE要求：

方法 DELETE
網址 https://api.bcovlive.io/v1/ssai/beaconsets/beaconset/beacon_set_id
標頭 X-API-KEY: your API KEY

樣品回复

響應將如下所示：

  {
    "beacon_set_id": "Beacon set ID",
    "deleted": true
  }
