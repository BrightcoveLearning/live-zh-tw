快速開始：使用Brightcove Live API創建直播

本教程將引導您逐步使用Live API創建實時流作業。

你需要什麼

要完成本教程，您需要以下內容。

  • 為Live API設置的帳戶
  • Live API的api密鑰（設置帳戶時提供給您）
  • Video Cloud帳戶（創建實時流作業不是必需的，但需要在Brightcove Player中設置播放）
  • 如果您將使用內容管理系統 API要創建 Video Cloud 視頻而不是 Studio，您需要具有視頻/讀取和視頻/寫入權限的客戶端憑據 - 如果您不熟悉獲取客戶端憑據的過程，請參閱驗證以下部分。
  • 可以通過以下方式發出REST API請求捲曲或REST客戶端（例如失眠或者郵差。

    為了幫助您，我們準備了一些可用於練習的失眠症和Postman進口品（在Brightcove Learning Services，我們更喜歡並使用Insomnia，但是Postman也會在您擁有的情況下工作）：

    在每個zip文件中，您都可以找到一個自述頁面來幫助您。

    • 失眠工作區導入
    • 郵遞員收集和環境導入
  • 要創建實際的實時流，您將需要用於實時流的軟件或硬件編碼器。下面的說明將介紹使用Wirecast設置直播的步驟（報名參加試用）。如果您使用其他編碼器，則過程將有所不同，但就Live API響應所需的信息而言，該過程應相似。

在大多數情況下，獲取憑據的最簡單方法是通過Studio Admin API身份驗證部分（需要您帳戶的管理員權限）。看管理API身份驗證憑據有關詳細信息。在這種情況下，您需要的權限是分享關係 -您同時需要讀寫權限：

如果Studio中沒有所需的權限，或者您希望直接從OAuth API獲取權限，請選擇獲取客戶憑證下面列出的文件。無論選擇哪個選項，都需要詢問正確的操作權限。以下內容可以與cURL或Postman一起使用以獲取適當的權限：

    "video-cloud/videos/read",
    "video-cloud/videos/create",
    "video-cloud/videos/update"]

設置視頻雲視頻

現在我們有了實時流信息，我們將在Video Cloud中設置一個遠程視頻，以便我們可以在Brightcove Player中播放它。

在這些步驟中，我們使用CMS API，但是如果您願意，也可以在Video Cloud Studio中完成-請參閱下面的替代步驟。

如果您使用的是Insomnia或Postman，則需要使用客戶端憑據設置OAuth2身份驗證。如果需要幫助，請參閱以下指南之一：

  • 對API請求使用失眠
  • 使用Postman進行API請求

  1. 首先，您需要通過製作一個新視頻POST請求：

    https://cms.api.brightcove.com/v1/accounts/account_id/videos

  2. 請求主體可以包含幾段元數據（請參見API參考（有關詳細信息），但唯一需要的是name：

    {
    "name": "some name for your live stream"
}

  3. 響應將包含所有視頻元數據，但重要的字段是id，您將需要添加遠程資產。

  4. 要添加HLS流，您將創建另一個POST請求：

    https://cms.api.brightcove.com/v1/accounts/account_id/videos/video_id/assets/hls_manifest

  5. 這次，請求正文將為：

    {
    "remote_url": "playback_url for your live stream"
}

使用有線廣播流式傳輸事件

在本節中，我們將設定並使用 Telestream Wirecast 編碼器來支援即時串流事件。Telestream Wirecast 軟體是一個桌面應用程式，可以從您的相機擷取輸入，並產生可以由 CDN 傳遞的資料流。請注意，有些硬體和其他軟體的編碼解決方案可能更適合用來傳遞您的直播活動。您可以從以下網址下載Wirecast軟件的試用版： Telestream網站。

注意：在繼續操作之前，您應該已經安裝了Wirecast軟件並已將相機連接到計算機。

要為現場直播配置Wirecast，請按照以下步驟操作。

  1. 開啟「線上轉播」
  2. 將鼠標懸停在+，點擊捕獲設備然後選擇您的相機。
  3. 單擊箭頭按鈕 ( ) 使相機拍攝實況鏡頭。
  4. 點擊溪流按鈕 （ ）。系統將提示您配置輸出設置。
  5. 設置目的地至RTMP服務器。點擊好的。
  6. 為了地址， 使用stream_url從Live API返回（第3步）。
  7. 為了溪流，活著使用。
  8. （可選的）要創建其他輸出流，請單擊添加...創建其他輸出流時，您可能希望創建自己的自定義編碼配置文件以控制輸出比特率。確保溪流名稱反映了新的比特率。
  9. 點擊好的保存輸出設置。
  10. 點擊溪流按鈕 （ ）。現場直播應該開始。

結論

現在，您可以創建實時流會話了。您需要在編碼器配置中添加以下內容：

  • stream_url：添加stream_url從你的工作中rtmp地址）
  • stream_name：流名稱必須為alive

啟動實時流後，您應該會看到它出現在前面步驟中創建的Brightcove播放器中。
