    使用 Akamai 串流封裝和電流線控制器建立即時串流

    在本主題中，您將學習如何使用Akamai Stream Packaging和Telestream Wirecast直播直播事件。

    本主題將幫助您使用Akamai Stream Packaging直播直播活動，  Telestream Wirecast編碼器，遠程資產視頻和Brightcove Player。也可以使用其他編碼器。如需 Brightcove 提供即時串流視訊的所有可用選項清單，請參閱提供即時串流。

    要使用Akamai Stream Packaging直播直播，您將：

    • 啟用您的視頻雲帳戶進行實時流式傳輸
    • 使用媒體模塊創建實時流視頻
    • 為直播活動配置Telestream Wirecast
    • 為現場活動生成發布代碼

    為您的視頻雲帳戶啟用實時流式傳輸

    在進行直播活動之前，您應該聯繫Brightcove支持並告訴他們您希望您的視頻雲帳戶啟用實時流式傳輸。還要求為該帳戶啟用流打包。配置帳戶後，您將收到一封電子郵件，其中包含設置實時流媒體所需的一組憑據和配置信息。

    注意：Akamai HD的憑據根據您的編碼器IP在地理位置上進行了設置。如果您計劃在新的位置舉辦活動，則應為Brightcove支持人員提供新位置的編碼器的外部可ping IP地址。這樣，Akamai可以在盡可能少的公共互聯網上旅行，從而在盡可能近的位置重新設置入口點，以減少延遲和中斷風險。Brightcove支持人員需要提前了解這一點，因為更改需要一些時間才能在Akamai的網絡中傳播。如果編碼器還不在該位置，那很好，只要網絡上有一個外部可ping通的IP地址即可使用。

    使用媒體模塊創建實時流視頻

    您必須在視頻庫中創建帶有遠程資產的視頻才能處理實時流。當您在自己或第三方系統上維護視頻文件，然後在創建視頻時向Brightcove提供視頻文件的URL時，將使用遠程資產。遠程資產是使用媒體模塊或使用CMS API。

    請按照以下步驟使用“媒體”模塊創建遠程資產視頻。

    1. 打開媒體模塊。
    2. 請點擊+添加遠程視頻在側面導航面板中。
    3. 輸入「視訊名稱」。
    4. 設置影片類型至生活。
    5. 設置格式至HLS。
    6. 為了渲染網址，從ARL Brightcove支持部門在您的電子郵件中提供的價值。ARL URL的格式為： 
       http://bcliveunivsecure-lh.akamaihd.net/[EVENT_ANGLE_BITRATE]@684348
      您必須更換事件和角度具有值的佔位符。比特率將不會被使用。例如，您的URL可能是： 
       http://bcliveunivsecure-lh.akamaihd.net/Live_1@684348

      插入一世在網址之後akamaihd.net網址的一部分：

       http://bcliveunivsecure-lh.akamaihd.net/i/Live_1@684348

      另外，附加/master.m3u8到URL的末尾。最終呈現URL應該類似於：

      http://bcliveunivsecure-lh.akamaihd.net/i/Live_1@684348/master.m3u8
    7. 將格式化的URL粘貼到網址字段並單擊添加渲染。
    8. 請點擊保存保存遠程資產視頻。
    9. 確認遠程資產出現在“媒體”模塊中。啟用遠程資產視頻（如果需要）。

    在“媒體”模塊中創建遠程資產視頻之後，您可以像其他任何視頻一樣編輯其元數據。

    為直播活動配置Telestream Wirecast

    在本節中，我們將設定並使用 Telestream Wirecast 編碼器來支援即時串流事件。Telestream Wirecast 軟體是一個桌面應用程式，可以從您的相機擷取輸入，並產生可以由 CDN 傳遞的資料流。請注意，有些硬體和其他軟體的編碼解決方案可能更適合用來傳遞您的直播活動。您可以從以下網站下載Wirecast軟件的試用版： Telestream網站。

    注意：繼續進行之前，您應該安裝 Wirecast 軟體，並將相機連接至您的電腦。

    若要設定 Wirecast 的即時事件，請依照下列步驟執行。

    1. 開啟「線上轉播」
    2. 按一下 + 並選取 [ 視訊擷取]。
    3. 從清單中選取您的相機。按一下「新增」。
    4. 確認選取的視訊來源顯示在預覽窗格中。
    5. 點擊箭頭按鈕（），以使相機拍攝到實況鏡頭。
    6. 按一下串流按鈕 ( )。系統會提示您輸入輸出設定。
    7. 將 [ 目的地 ] 設定為 RTMP 伺服器。按一下「確定」。
    8. 為了地址， 使用主要入口點 Brightcove支持人員在電子郵件中提供的價值。
    9. 為了流，請使用在定義遠程資產URL時使用的[EVENT_ANGLE_BITRATE] @ 394443值。確保為創建遠程資產時使用的EVENT和ANGLE使用相同的值。將BITRATE值設置為Wirecast中設置的比特率。
    10. 請點擊設置憑據...
    11. 輸入用戶名和密碼由Brightcove支持人員在電子郵件中提供。按一下「確定」。
    12. 按一下「確定」以儲存輸出設定。
    13. 點擊信息流按鈕。現場直播應該開始。

    為現場活動生成發布代碼

    1. 打開媒體模塊。
    2. 選擇實時流遠程資產視頻。
    3. 按一下「發佈和嵌入」。
    4. 選擇一個球員。
    5. 單擊播放器URL，以在瀏覽器中預覽實時流。

    如需進一步說明

    如果您需要幫助以使現場活動開始進行，則可以使用Brightcove支持。您可以提交案件這裡。為了確保您能夠獲得最快速的回應，以下是 Brightcove 支援部門解決問題所需的清單。

    • 流的具體症狀。例如，它是根本不播放還是口吃或凍結？
    • 這個流是否在過去正常工作
    • 您在編碼器中使用的入口點 URL
    • 流名稱
    • 您使用的編碼軟件和硬件
    • 您發佈直播活動的玩家 URL
    • 您的實時遠程資產的視頻ID
    • 從編碼器到發佈點主機的追蹤路由結果
