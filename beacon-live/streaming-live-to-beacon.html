    將即時事件串流至信標應用程式

    在本主題中，您將學習如何將即時活動串流至 Beacon 應用程式。

    概覽

    若要將即時活動串流至您的應用程式，您必須遵循一系列步驟來連接您的視訊雲端 Live 帳戶和布萊特灣信標。

    該過程的高級視圖是：

    1. 第一步是創建一個事件和一個播放列表在布萊特灣信標。此活動將在應用程序中作為您的視頻佔位者提供。
    2. 移至您的視訊雲端帳戶並存取 Live 以建立您的事件，並取得可作為識別碼的工作 ID。
    3. 回到 Brightcove Beacon，您將能夠連接您的即時串流，將即時平台中產生的工作 ID 放在平台的「事件」區段下。
    4. 等待連線，即時視訊會在 Beacon 應用程式中開始串流。
    5. 在最後一節中，你會看到通過靜態入口點創建一個不同類型的事件在現場的步驟。

    本文件的其餘部分將提供每個步驟的詳細資料。

    在信標 (Beacon) 中建立即時事件：

    在Brightcove信標平台中，單擊大事記標籤以啟動事件的設置：

    只有在 B rightcove Beacon 直播啟動的帳戶才能在標籤頁中顯示播放清單和活動。如果您沒有啟動，請聯繫布萊特灣支援部門。
    1. 點擊新增活動：
    2. 給直播活動一個名稱， 和活動開始日期：
    3. 在裡面文字數據標籤，填寫您的活動信息：
    4. 在裡面非文字數據標籤，為您的活動選擇圖片海報：

      您可以在「縱向」、「橫向」、「縮圖」和「寬廣橫幅」之間選擇您想要的海報類型。

    5. 在裡面權利和計劃標籤，您將可以設置可用性詳細信息如：
      • 權限類型：選擇「 廣告」 、「 訂閱」 或「 免費」之間的權利類型 。
      • 開始時間：活動的開始時間
      • 時間結束：事件的結束時間，這些時間會建立在應用程式內產生限制。
      • 最大流：是應用程序允許的最大流數。預設數字為 0 (零)。
      • 設備詳細信息：在這裡，您可以添加您想要發布串流的不同數量的設備。（例如：iOS，安卓，六，三星等）
      • 地理位置詳細信息：您希望啟用應用程式的國家/地區清單。如果你不想限制任何國家，你可以選擇「世界」。
      • 拒絕的位置：拒絕存取您的內容的地區清單。
    6. 點擊以下按鈕：建立新活動

    將信標事件新增至播放清單

    創建事件後，請轉到播放清單上方菜單中的標籤。我們需要將事件添加到播放列表之前，我們才能串流某些內容。

    1. 點擊新增播放清單按鈕並創建一個。
      您需要在布萊特灣信標中建立一個包含您的直播活動的播放清單。不要使用Video Cloud中的播放列表，使用Video Cloud中的播放列表可能會覆蓋並刪除您的Beacon Live錄製內容。
    2. 在播放清單，從內容類型標籤中，選擇大事記標籤：
    3. 在內容模組中尋找您的事件，然後按一下 (+) 按鈕，將其新增至左側播放清單。您可以按照您希望在應用程式中顯示事件的方式來定位事件的順序。
    4. 調整播放清單的設定 (時間、裝置、位置等...)就像你的活動一樣
    5. 按一下「儲存播放清單中的變更」按鈕。
    6. 點擊屏幕右上角的工具按鈕以清除緩存，以便在應用程序中正確顯示所有內容。

    此時，您的活動將被創建為 Brightcove Beacon 應用程序中的容器，但要實際上開始串流直播視頻，我們需要在視頻雲端直播平台中設置我們的傳輸。

    在工作室創建一個直播活動

    若要存取即時模組，請登入影片雲端 Studio，然後按一下導覽標題中的 [即時]。

    1. 從「管理事件」頁面中，按一下「建立事件」。「建立即時事件」或「建立即時頻道」頁面隨即開啟。建立事件和通道時的屬性是相同的。本文件中的螢幕擷取畫面涵蓋建立事件。
    2. 輸入活動詳細資料：
      • 事件名稱 -事件的名稱；如果視訊儲存為視訊資產，則此名稱也會出現在「媒體」模組中 (最多 255 個字元)
      • 簡短說明 -事件的摘要 (最多 250 個字元)
      • 標籤 -要與事件關聯的標籤；如果視訊儲存為視訊資產 (最多 128 個字元和 1200 個標籤)，則會將標籤指派給視訊
      • 參考 ID -事件的參考 ID；必須是唯一的 (最多 150 個字元)
    3. 將顯示帳戶的任何必要自訂欄位。將值指派給自訂欄位。
    4. 將「串流至社交媒體」設定為「允許」，活動可以串流至社交媒體平台。可以做到一旦事件開始流式傳輸或使用社會模塊。
    5. 選取「即時內嵌設定檔」。下拉式清單會列出所有即時內嵌設定檔。內嵌描述檔將控制所建立的即時轉譯的品質和數目。按一下「管理設定檔」連結將會在其他瀏覽器標籤中開啟內嵌設定檔編輯器。請注意，您必須是帳戶管理員才能存取內嵌設定檔編輯器。按一下轉譯連結，檢視將使用選取的內嵌輪廓建立的轉譯。
      三個層級的位元率和帳單詳細資料如下：
      • 第 1 級允許高達 5,500 Kbps，並以 1.0 小時計費
      • 第 2 層允許高達 10,000 Kbps，並以 1.5 小時計費
      • 第 3 級允許高達 20,000 Kbps，並以 2.0 小時的方式進行充電
    6. 選取 [ 完成時將事件轉換為視訊資產 ]，即可將直播影片的 VOD 版本儲存到您的 Video Cloud 帳戶。選取此選項後，會使用所選「即時內嵌設定檔」中的最高品質轉譯作為 VOD 視訊的來源。建立轉譯時，會使用帳戶的預設內嵌設定檔。請注意，即時活動結束後就無法使用 VOD 版本，視訊會在擷取程序完成後才能使用。
      注意：使用週期性事件時，不支援此選項。
    7. 按一下「進階選項」並設定進階選項。
      注意：若要使用「信標 (Beacon) 即時」工作，您必須啟用「進階」選項中的「串流狀態通知」
    8. 啟用直播狀態通知

      從信標 CMS 新增通知網址 (此 URL 將由布萊特灣支援部門提供)

    9. 按一下「建立事件」以啟動即時事件。[控制室] 頁面將會開啟，其中包含即時事件的詳細資料，包括編碼器將使用的串流端點 (RTMP URL) 和串流名稱。

      處理串流需要大約一分鐘的時間。

      若要在直播平台中開始串流，您需要編碼器，如需有關如何使用編碼器的資訊，請閱讀以下文件：

      逐步：使用即時模組與電流線上傳送

      逐步：使用開放式廣播軟體 (OBS) 的直播模組

    10. 在裡面控制室複製“工作ID”和“視頻ID”來自測試事件面板上，我們都需要在Beacon平台設置中配置Live事件。
    11. 接下來，返回信標並單擊事件選項卡，轉到我們之前在那裡創建的事件，並在基本數據設置中粘貼即時事件的作業 ID。

      然後去流標籤，然後在流視頻ID，粘貼影片編號從我們之前創建的視頻雲事件中獲得。

      請填寫以下資訊：

      • 選擇您可以在其中發佈視訊的裝置數目。
      • 改變流類型至：生活
      • 然後點擊“更新事件”離開頁面前的按鈕：
    12. 清除您的快取。

      前往布萊特灣信標應用程式並選擇您的播放清單和活動。點擊玩。

      您現在應該可以在 Beacon 應用程式中從視訊雲端查看您的即時串流。

      活動結束後，您可以將它儲存為剪輯片段，以便稍後以信標上的 VOD 形式傳送。

    從您的信標即時事件建立剪輯片段

    如果您想要將視訊保留在稍後以 VOD 錄製的視訊串流形式播放，您必須建立即時信標事件的剪輯片段。

    Live 模組提供從已完成的事件建立視訊剪輯的功能。您也可以在現場活動進行時建立剪輯片段。

    如需完整的剪裁功能清單，請參閱使用即時模組建立視訊片段的說明文件。

    1. 在實時模塊中，轉到創建剪輯左側面板中的標籤
    2. 使用剪裁按鈕或快速鍵，設定剪輯片段的起點和終點。或只需點擊保存按鈕保留整個視頻。
    3. 輸入剪輯視訊屬性。
      • 剪輯片段名稱 -剪輯片段的名稱
      • 簡短描述 -剪輯片段的說明
      • 標籤 -要與剪輯片段建立關聯的標籤
      • 參考 ID -指定給剪輯片段的參考 ID
      • 移除來源廣告中斷內容 -此選項會移除剪輯片段內觸發的廣告分段，只保留節目內容。注意：只有在串流啟用 SSAI 時，才會出現「移除來源廣告片段」核取方塊。
    4. 按一下「建立剪輯」。剪輯片段的轉碼程序將會開始。剪輯片段會使用帳戶的預設內嵌設定檔進行轉碼。

    將您的剪輯片段加入信標 (Beacon) 作為可播放的串流影片。

    1. 拿號碼ID您創建的剪輯：
    2. 在信標平台中，轉到大事記標籤，然後選擇流菜單，更改流類型在“實時”到“流”下拉菜單中流視頻ID選項，粘貼ID您剛剛從Live平台中的Clip事件複製的數字。
    3. 按一下「基本資料」標籤，並將狀態從「已結束」變更為「已完成」。離開之前，請點擊更新事件按鈕在頁面底部，這將使玩應用中的可用按鈕可查看事件的記錄。

    (靜態進入點) SEP 即時事件

    例如，當您的事件一遍又一遍地重新開始時，SEP 是可以讓您啟動/停用的週期性事件。此事件可以用作所有事件現場工作ID可以在信標應用程序中傳輸到多個通道。

    • 請注意，當您建立週期性事件時，會在「媒體」模組中自動為其建立視訊。請勿刪除此視訊，因為每次事件重複發生時都需要這個影片。
    • 在編碼器啟動之前，應該啟動週期性事件。

    若要建立週期性事件，請在建立即時事件時選取 [使用靜態進入點 (SEP) 建立這個為週期性事件] 選項，然後按一下 [ 建立事件 ]。

    1. 在直播平台中創建一個新活動。有關如何創建活動的更多信息，請閱讀以下內容文件資料。
    2. 展開高級選項，然後啟用“使用靜態入口點（SEP）將其創建為重複事件”選項。有關SEP的更多信息，請單擊這裡。
    3. 當您已經有SEP事件時，請點擊支持標籤，您將能夠激活/停用它來自實時控制室。
    4. 複製您的現場工作ID
    5. 返回布萊特灣信標平台並設定您的賽事名稱和排定時間。
    6. 活動的日期和時間必須採用下列格式：
    7. 按一下「更新事件」按鈕。

    您可以在 Brightcove Beacon 平台中使用您想要的所有事件頻道來執行此程序，並且所有這些通道都會從同一個即時事件傳輸中串流，但每個通道都有您在每個事件設定中調整的不同日期和時間。
